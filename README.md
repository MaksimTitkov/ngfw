# NGFW Manager v2.2

> Профессиональная система управления политиками безопасности для PT NGFW.  
> Централизованное управление несколькими системами управления, интеллектуальный перенос правил, анализ политик.

---

## Что нового в v2.2

### Синхронизация объектов между СУ (Cross-SU Object Sync)

Новая вкладка **Objects** на странице **Policy Diff** позволяет сравнивать объекты (сети, группы, сервисы) между двумя или более системами управления и синхронизировать отличия в один клик.

- Выберите две СУ — система покажет объекты, которые есть в одной и отсутствуют в другой
- Рядом с каждым расхождением — кнопка синхронизации: объект создаётся на целевой СУ с той же структурой
- Группы синхронизируются рекурсивно — вложенные члены создаются автоматически
- Результат немедленно записывается в локальный кэш — объекты доступны без повторной синхронизации

### Rule Sync — сравнение и массовое копирование правил между СУ

Новая вкладка **Rule Sync** на странице **Policy Diff** позволяет сравнивать правила безопасности между устройствами разных СУ и переносить их массово.

- Выберите устройство-источник и устройство-цель (разные СУ или одна)
- Система показывает правила, которые есть в источнике и отсутствуют на целевом устройстве
- Отметьте нужные правила и скопируйте все за одно действие
- При отсутствии папок на целевом устройстве — система предложит создать их автоматически
- Объекты правил разрешаются и создаются на целевой СУ через TransferService (тот же движок, что в обычном Transfer)

### Reverse Rule — создание зеркального правила

На странице правил в контекстном меню доступна команда **Reverse** — создаёт зеркальную копию правила с обменом источника и назначения.

- `sourceAddr` ↔ `destAddr` — адреса меняются местами
- `sourceZone` ↔ `destZone` — зоны меняются местами
- Зеркальное правило создаётся с суффиксом `_rev` в имени и в состоянии **disabled** по умолчанию
- Если правило `_rev` уже существует — добавляется числовой суффикс (`_rev_2`, `_rev_3`), дублирования нет
- Поддерживается для нескольких правил одновременно (bulk reverse)

### Умный анализ конфликтов при переносе и реверсе

При переносе правил (Transfer) и создании реверсных копий система теперь анализирует конфликты имён:

| Ситуация | Поведение |
|----------|-----------|
| Правило с таким именем уже существует и идентично | Пропускается, не создаётся дубль |
| Правило с таким именем существует, но отличается | Добавляется суффикс `_copy` / `_rev_N`, пользователь видит предупреждение |
| Правило уникально | Создаётся как обычно |

### Экспорт правил в JSON (raw формат СУ)

Новый пункт экспорта в разделе **Export** — выгрузка правил устройства в JSON-формате, идентичном формату API PT NGFW.

- Правила сгруппированы по секциям: `pre`, `default`, `post`
- Содержит оригинальные UUID объектов из кэша
- Удобно для отладки, аудита и ручной миграции

### Интерактивный аккордеон блоков в таблице правил

Блоки внутри папки теперь можно **сворачивать и разворачивать** прямо в таблице правил.

- Заголовок каждого блока показывает имя, подсеть VLAN и количество правил
- Состояние каждого блока сохраняется в сессии — перезагрузка страницы не сбрасывает раскладку
- Правила без блока отображаются в отдельной секции **General** в конце списка

### Создание правила прямо в блоке

Рядом с каждым заголовком блока добавлена кнопка **+** — открывает форму создания правила с уже выбранным блоком. Не нужно выбирать блок вручную после открытия модального окна.

### Автоматическая проверка подсети при добавлении правила в блок

Если IP-адреса источника или назначения добавляемого правила не соответствуют подсети блока — система предупреждает и предлагает три варианта:

| Вариант | Действие |
|---------|----------|
| Рекомендованный блок | Переместить правило в блок, подсеть которого совпадает |
| Оставить в текущем блоке | Добавить как есть, без изменений |
| General | Добавить правило без привязки к блоку |

Проверка работает рекурсивно — группы объектов раскрываются до IP-уровня (глубина до 5 уровней).

## Что нового в v2.1

### IP Sorter — сортировка правил по IP-плану

**IP-план** — это таблица соответствия VRF → VLAN → подсеть для каждого устройства. На основе IP-плана Sorter автоматически классифицирует правила и предлагает структуру блоков.

#### Как подключить IP-план

1. Перейдите в **SYS** → вкладка **IP Plan**
2. Выберите нужное устройство в сайдбаре
3. Загрузите файл (CSV или XLSX) — формат A: `vrf, vlan, subnet, [mask], [description]`
4. Либо добавляйте записи вручную кнопкой **Добавить строку**

> IP-план хранится строго для каждого устройства отдельно. Данные зашифрованы в БД через AES-128 (Fernet).

#### Как использовать Sorter

1. Загрузите IP-план для устройства (шаг выше)
2. Перейдите в **SEC → Sorter** (`/sort-analysis`)
3. Выберите устройство — система покажет таблицу классификации правил:
   - **Classified** — правило сопоставлено с конкретным VRF/VLAN/подсетью
   - **Unclassified** — адреса правила не попали ни в одну подсеть плана
4. Просмотрите предложенную структуру папок и блоков
5. Примените вручную или настройте автоматический запуск через планировщик

#### Автоматический анализ по расписанию

1. Перейдите в **SYS → Scheduled Tasks**
2. Нажмите **+ Add Task**
3. Выберите тип задачи **Sort Analysis**, укажите устройство и интервал
4. Результаты появятся в логах приложения (`classified=N unclassified=M`)

#### Блоки внутри папок

После применения Sorter-предложения правила группируются в блоки. В главной таблице правил блоки отображаются с интерактивными заголовками-аккордеонами (добавлено в v2.2).

## Что нового в v2.0

### Управление несколькими системами управления (СУ)

В v1.x поддерживалась только одна СУ. В v2.0 можно подключить **любое количество систем управления** PT NGFW и работать с ними из единого интерфейса:

- Все СУ отображаются в боковой панели с группировкой по системе
- Переключение между СУ — один клик
- Учётные данные каждой СУ хранятся в зашифрованном виде в базе данных

### Межсистемный перенос правил (Cross-SU Transfer)

Главная фича версии 2.0 — **интеллектуальный перенос правил между разными СУ**:

| Сценарий | Поведение |
|---|---|
| Объект есть на целевом устройстве | Найден по fingerprint (IP/CIDR/протокол+порты) и переиспользован |
| Объект отсутствует на целевом устройстве | Автоматически создаётся копия с уникальным именем |
| Глобальный объект, та же СУ | UUID переиспользуется напрямую |
| Глобальный объект, другая СУ | Создаётся устройство-специфичная копия |
| Группа объектов | Рекурсивное разрешение всех членов, создание группы |
| Вложенные группы | Поддерживается любая глубина вложенности |

Все созданные объекты немедленно кэшируются в базе — правила отображаются корректно без дополнительной синхронизации.

### Предкомпилированные модули

Ключевые модули поставляются в виде скомпилированных нативных расширений Python (`.so`): клиент NGFW API, движок синхронизации, движок переноса правил и анализатор политик. Это обеспечивает стабильную работу независимо от окружения и не требует дополнительной сборки при деплое.

### Изолированное хранилище объектов

Все объекты хранятся с изолированными идентификаторами `{su_id}:{raw_uuid}` — исключены коллизии ID между разными СУ даже при использовании клонированных систем с одинаковыми именами устройств.

## Требования

- **Docker** 24+ и **Docker Compose** v2
- **Linux** x86_64 (скомпилированные .so требуют x86_64)
- Сетевой доступ до API PT NGFW (порт 443)

> Версия Python внутри контейнера: **3.12**

## Быстрый старт

### 1. Клонировать репозиторий

```bash
git clone <repo-url>
cd ngfw-manager
```

### 2. Настроить окружение

```bash
cp .env.example .env
```

Откройте `.env` и заполните:

```env
APP_TITLE="NGFW Manager"
SECRET_KEY=<случайная строка 32+ символа>
DB_PASSWORD=<надёжный пароль>
```

Сгенерировать SECRET_KEY:

```bash
python3 -c "import secrets; print(secrets.token_hex(32))"
```

### 3. Настроить SSL

```bash
bash init_ssl.sh
```

Создаст самоподписанные сертификаты в `nginx/certs/`.  
Для продакшена положите туда `ngfw.crt` и `ngfw.key`.

Раскомментируйте порт 443 в `docker-compose.yml`:

```yaml
ports:
  - "80:80"
  - "443:443"   # ← раскомментировать
```

### 4. Запустить

```bash
docker compose up -d --build
```

Первый запуск занимает 2–4 минуты (сборка Vue-фронтенда + установка зависимостей).

Приложение доступно по адресу: **https://your-server**

### 5. Первый вход

Откройте браузер → введите адрес сервера.

На странице входа:
- **NGFW Host** — адрес вашей СУ, например `https://10.1.31.100`
- **Username** / **Password** — учётные данные API PT NGFW

После входа будет автоматически запущена синхронизация устройств.

---

## Управление несколькими СУ

После первого входа система создаёт запись СУ. Для добавления второй СУ:

1. Перейдите в **SYS** → раздел управления системами
2. Нажмите **+ Add Connection**
3. Введите адрес, логин, пароль второй СУ
4. Запустите синхронизацию

Устройства обеих СУ появятся в боковой панели с группировкой.

## Основной функционал

### Правила безопасности

- Просмотр правил Pre / Default / Post для любого устройства любой СУ
- **Виртуальные папки** — локальная организация правил без изменения NGFW
- Drag-and-drop сортировка правил и папок
- Создание правил через форму с пикером объектов
- Редактирование, включение/выключение, смена действия
- **Bulk-операции**: включить/выключить, изменить action, переместить, удалить, сохранить как шаблон
- **Transfer / Copy** — перенос правил между устройствами и СУ
- **Deploy** — применение порядка правил к NGFW через API

### Policy Analyzer

Автоматический анализ политик с обнаружением:

| Тип проблемы | Описание |
|---|---|
| Теневые правила | Правило перекрыто более широким выше в списке |
| Избыточные правила | Дублирующиеся правила с одинаковым эффектом |
| Слишком широкие | Правила с Any/Any/Any без ограничений |
| Отключённые | Правила в состоянии disabled |

Анализ запускается автоматически после синхронизации.

### Объекты

- Полный CRUD: сетевые объекты, группы, сервисы, группы сервисов
- **Двухуровневый селектор**: СУ → устройство (не запутаетесь между клонами)
- Object Usage Map — поиск всех правил, использующих объект
- Bulk-удаление

### Другие возможности

- **Smart Search** — поиск по IP, CIDR, порту, имени (Ctrl+K)
- **Change Log** — журнал всех изменений с фильтрами
- **Policy Diff** — сравнение политик двух устройств
- **Rule Templates** — библиотека шаблонов правил
- **NAT Rules** — просмотр и редактирование NAT-политик
- **Traffic Logs** — просмотр логов трафика с фильтрами
- **Экспорт в Excel** — правила с цветовой кодировкой
- **Экспорт в YAML** — структура виртуальных папок
