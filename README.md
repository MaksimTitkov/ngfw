# NGFW Manager

**Версия: 1.1 (stable)**

Веб-интерфейс для управления политиками безопасности NGFW.  
Позволяет организовывать правила в виртуальные папки, создавать, редактировать и переносить правила между устройствами — не затрагивая внутреннюю структуру NGFW.

---

## Возможности

| Раздел | Описание |
|--------|----------|
| Security Rules | Просмотр, создание, редактирование, удаление правил. Виртуальные папки (Pre/Default/Post). Drag-and-drop сортировка. Bulk-операции |
| NAT Rules | Просмотр и редактирование NAT-правил. Виртуальные папки. Deploy |
| Objects | Полный CRUD сетевых объектов, групп, сервисов, групп сервисов. Object Usage Map |
| Logs | Просмотр логов трафика с фильтрами |
| Policy | Decryption / Auth / PBR правила |
| System | Статус системы, device groups |
| Dashboard | Сводная страница: счётчики, статус устройств, последние изменения, алерты |
| Policy Analyzer | Обнаружение теневых, избыточных, слишком широких и отключённых правил |
| Change Log | Журнал всех изменений с фильтрацией по пользователю, типу и дате |
| Policy Diff | Сравнение политик между двумя устройствами. Просмотр pending-изменений |
| Smart Search | Поиск правил по IP, CIDR, порту или имени. Ctrl+K overlay |
| Rule Templates | Сохранение правил как шаблонов, применение на любое устройство/папку |
| Export | Экспорт правил в Excel (.xlsx), структуры папок в YAML |

**Интерфейс:** доступен на двух языках — русский и английский (переключение в боковой панели).

## Требования

- Docker 24+ и Docker Compose v2
- Сетевой доступ к NGFW Management API (HTTPS)
- Порты 80 и 443 (или другие — настраиваются в `docker-compose.yml`)

## Быстрый старт

### 1. Клонировать репозиторий

```bash
git clone https://github.com/MaksimTitkov/ngfw.git
cd ngfw
```

### 2. Настроить окружение

Отредактируйте `.env` при необходимости (файл уже есть в корне):

```env
APP_TITLE="NGFW Manager"
```

### 3. Сгенерировать SSL-сертификаты

```bash
bash init_ssl.sh
```

Создаст самоподписанные сертификаты в `nginx/certs/`.  
Если у вас есть валидные сертификаты — положите их туда как `ngfw.crt` и `ngfw.key`.

### 4. Запустить

```bash
docker compose up -d --build
```

При первом запуске Docker скачает образы и соберёт контейнер — это займёт 1–2 минуты.

Приложение доступно по адресу: **https://localhost**

### 5. Остановить

```bash
docker compose down
```

## Вход в систему

Откройте **https://localhost** в браузере (или IP/домен сервера).

На странице входа:
- **NGFW Host / URL** — адрес вашей системы управления, например `https://192.168.1.100`
- **Username** — имя пользователя NGFW API
- **Password** — пароль

Сессия сохраняется в cookie браузера.

## Основной функционал

### Синхронизация (Sync)

Кнопка **Sync** в верхней панели загружает с устройства:
- Все device groups
- Все объекты (сети, сервисы, зоны, приложения, пользователи)
- Все правила безопасности и NAT

Повторная синхронизация обновляет кэш. Рекомендуется выполнять после изменений на стороне NGFW.

### Виртуальные папки

В боковом сайдбаре:
- Создать папку — кнопка **+ Add Folder** под списком устройства
- Перетащить правила — drag-and-drop
- Переместить несколько правил — выделить чекбоксами → **Move to Folder** в bulk-bar

Структура папок хранится только локально в БД — NGFW её не видит.

### Создание правила

Кнопка **+ New Rule** открывает форму:
- Название, описание, действие (Allow / Drop / Deny)
- Зоны источника и назначения
- Сетевые объекты и сервисы
- Enabled / Disabled

Объекты выбираются через пикер с живым поиском (мультиселект).

### Bulk-операции

Выделите несколько правил чекбоксами. В нижней панели доступны:
- **Enable / Disable** — включение/выключение
- **Set Action** — смена действия (Allow/Deny/Drop) для группы
- **Logging** — включить/выключить логирование
- **Move to Folder** — переместить в другую папку
- **Transfer / Copy** — скопировать или перенести на другое устройство
- **Template** — сохранить как шаблон
- **Delete** — удалить

### Deploy

Кнопка **Deploy** применяет текущий порядок правил из виртуальных папок к NGFW.

### Policy Analyzer

Раздел **ANLY** — анализ кэша правил без запросов к NGFW:
- **Теневые (Shadowed)** — правило перекрыто более ранним и никогда не сработает
- **Слишком широкие (Too Broad)** — Allow с Source=Any и Destination=Any
- **Избыточные (Redundant)** — соседние правила с одинаковым действием, зонами и сервисом
- **Отключённые (Disabled)** — правила с `enabled=false`

### Smart Search (Ctrl+K)

Быстрый поиск доступен на любой странице (Ctrl+K).  
Раздел **SRCH** — расширенный поиск с режимами:
- **Any** — поиск по имени + IP + порту
- **IP** — поиск правил, где задан конкретный IP или CIDR
- **Port** — поиск по порту назначения
- **Name** — поиск по имени правила (частичное совпадение)

### Change Log

Раздел **CLG** — журнал всех изменений.  
Фильтры: текст, тип действия (create/update/delete/toggle), тип объекта, пользователь.

### Policy Diff

Раздел **DIFF**:
- **Device Comparison** — сравнение правил двух device-групп (кто есть в A, чего нет в B, что отличается)
- **Modified (Pending Deploy)** — правила, изменённые локально, но ещё не применённые к NGFW

### Rule Templates

Раздел **TMPL** — библиотека шаблонов.  
Сохранение: открыть правило → выделить → **Template** в bulk-bar.  
Применение: выбрать шаблон → **Apply to Folder** → выбрать устройство и папку → **Apply**.

### Export

В разделе Security Rules (кнопка **Export**):
- **Excel (.xlsx)** — таблица правил с цветовой кодировкой action и именами объектов
- **YAML** — снапшот структуры папок с порядком правил

## Обновление

```bash
# Остановить
docker compose down

# Обновить код
git pull origin main

# Пересобрать и запустить
docker compose up -d --build
```

> Если обновлялась схема БД — после запуска рекомендуется выполнить повторный **Sync**.
