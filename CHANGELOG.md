## v1.1 (stable)

> Стабильная production-версия.

### Новые разделы и страницы

| Раздел | URL | Описание |
|--------|-----|----------|
| Dashboard | `/dashboard` | Сводная страница: счётчики правил/объектов, статус устройств, список последних изменений, алерты, быстрая навигация |
| Smart Search | `/search` | Поиск по IP, CIDR, порту, имени правила. Режимы: Any / Name / IP / Port. Поддержка Ctrl+K overlay на всех страницах |
| Policy Analyzer | `/analyzer` | Локальный анализ кэша правил: теневые, слишком широкие, избыточные, отключённые правила |
| Change Log | `/changelog` | Журнал всех изменений: кто/что/когда. Фильтры по дате, пользователю, типу, действию. Постраничная загрузка |
| Policy Diff | `/diff` | Сравнение политик двух device-групп; просмотр локально изменённых правил (pending deploy) |
| Rule Templates | `/templates` | Библиотека шаблонов: сохранение правила как шаблона, применение в любую папку любого устройства, bulk-удаление |

### Управление объектами (полный CRUD)

- **Создание сетевых объектов** — Host/Network, CIDR, FQDN, IP-range через модальную форму
- **Редактирование объектов** — двойной клик → Edit-модалка с предзаполнением
- **Удаление объектов** — одиночное и bulk (с подтверждением)
- **Сетевые группы** — Create / Update / Delete членов группы
- **Сервисы и группы сервисов** — полный CRUD (TCP, UDP, ICMP, группы)
- **Object Usage Map** — кнопка «Find in Rules» для любого объекта → поиск всех правил, где он используется

### Правила безопасности

- **Массовые операции** (bulk-bar):
  - Включение / выключение (enabled/disabled)
  - Изменение action (Allow / Deny / Drop) для группы правил
  - Включение логирования для выбранных правил
  - Перемещение в другую папку
  - Удаление нескольких правил
  - Сохранение как шаблон
- **Экспорт в Excel** (`GET /api/v1/export/rules/xlsx`) — цветовая кодировка action, auto-filter, имена объектов
- **Экспорт структуры папок в YAML** (`GET /api/v1/export/folders/yaml`) — полный snapshot виртуальной структуры

### NAT правила

- **Редактирование NAT-правила** — двойной клик по строке → Edit-модалка, `POST /api/v1/nat/rules/update`

### API

- `POST /api/v1/objects/update` — обновление сетевых объектов и групп
- `POST /api/v1/nat/rules/update` — обновление NAT-правил
- `POST /api/v1/search` — JSON API быстрого поиска (для Ctrl+K overlay)
- `GET /api/v1/object/usage` — поиск правил по ext_id объекта
- `GET /api/v1/export/rules/xlsx` — экспорт в Excel
- `GET /api/v1/export/folders/yaml` — экспорт структуры папок
- `POST /api/v1/templates/save` — сохранить шаблон
- `POST /api/v1/templates/apply` — применить шаблон к папке
- `POST /api/v1/templates/delete` — удалить шаблоны
- `POST /api/v1/analyzer/run` — запуск анализа политики
- `POST /api/v1/changelog/query` — фильтрованный запрос журнала
- `POST /api/v1/diff/devices` — сравнение правил двух устройств
- `GET /api/v1/diff/modified` — список локально изменённых правил

### БД

- Новая таблица `rule_templates` (хранение шаблонов локально)
- Новая таблица `change_log` (журнал изменений: user, action, entity_type, entity_id, detail, timestamp)

### Зависимости

```
openpyxl   — экспорт в Excel
pyyaml     — экспорт в YAML
```

## v1.0

> Базовая версия.

### Реализовано

- Авторизация через NGFW API (session cookie)
- Синхронизация device groups, объектов, правил безопасности (Pre/Default/Post)
- Виртуальные папки — локальная организация правил, не затрагивающая структуру NGFW
- Drag-and-drop сортировки правил и папок
- Создание правил безопасности (`POST /api/v1/rules/create`)
- Копирование / перенос правил между устройствами и папками (Transfer/Copy) с автоматическим клонированием объектов
- Просмотр и поиск объектов (сети, сервисы, зоны, приложения, пользователи)
- NAT правила — просмотр, виртуальные папки, deploy
- Policy (Decryption / Auth / PBR) — базовый просмотр
- Логи трафика — просмотр с фильтрами (Traffic Logs)
- Deploy — применение порядка правил к NGFW через `MoveSecurityRule`
- System / Admin — базовый просмотр
- Nginx HTTPS-прокси, Docker Compose, PostgreSQL 15
